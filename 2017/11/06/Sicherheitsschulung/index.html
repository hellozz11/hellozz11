Sicherheitsschulung

2017/11/06

This article was published before 337 days, and the content may be outdated.

  • E-Mail-Sicherheit

    • Phishing-E-Mail
    • E-Mail-Fälschung
    • Rollover von Drittanbietern
    • Überprüfen Sie den Absender
    • Aktivieren Sie die sekundäre Überprüfung
    • Mailweiterleitung
    • Sammlung von Dritten
    • E-Mail-Anhang sensible Informationen Verschlüsselung

  • Virenschutz
    Was ist ein Trojaner?

    • Welche Antivirensoftware installiere ich?
    • Aktualisieren Sie regelmäßig die Virendatenbank

  • Browsersicherheit

    • Durchsuchen Sie keine schädlichen Websites
    • Laden Sie keine Software von bösartigen Websites herunter
    • Software zum Herunterladen von der offiziellen Website herunterladen

  • Offenlegung von Informationen

    • Upload vertraulicher Daten in Cloud Notes, GitHub
    • Gemeinsames Konto
    • Verwenden Sie eine Firmen-E-Mail, um ein externes oder privates Konto zu registrieren
    • Social Network Publishing Company Technologielösungen und vertrauliche Produktinformationen
    • Firmenkonto und privates Konto sollten isoliert sein
    • Nehmen Sie es nach dem Drucken weg

  • PC-Sicherheit

    • Bildschirm sperren
    • Nicht einfach seltsame U-Disk einfügen
    • Rechtzeitige System-Updates
    • Laden Sie die Cracking-Software nicht von außen herunter (bitte suchen Sie ggf. IT-Studenten)
    • Nimm deine Firmendaten nicht vom Arbeitsplatz mit
    • Der Computer sollte ein Startkennwort hinzufügen, um ein Datenleck nach dem Verlust des Computers zu verhindern.
    • Enterprise WeChat-, WeChat-, QQ-, QQ-Gruppendatei, die Dateien zum Verpacken und Verschlüsseln sendet
    • Arbeitsdokument-Verschlüsselungsdiskette (mit Verschlüsselungssoftware wie TrueCrypt)

  • Passwort ist durchgesickert

    • Laut der Umfrage gibt es derzeit weltweit 80 Milliarden Passwörter und persönliche Privatsphäre.
    • Passwörter verwenden mehr als 12 Ziffern mit Sonderzeichen, Groß- und Kleinbuchstaben und Zahlen
    • Passwörter sollten nicht mit unternehmensinternen Passwörtern verknüpft sein
    • Das Passwort muss alle drei Monate geändert werden

  • WiFi-Sicherheit

    • Schließen Sie kein nicht vertrauenswürdiges WLAN an
    • Sie müssen sich nach dem Verbinden mit nicht vertrauenswürdigem WLAN mit dem VPN verbinden
    • Überbrücken Sie nicht privat im Arbeitsbereich, teilen Sie Büronetzwerk WIFI
    • Verwenden Sie keine Passwort-App mit gemeinsamem Passwort, z. B. WIFI-Masterschlüssel

  • VPN- und OA-Sicherheit

    • VPN-Konto und OA-Konto werden nur von sich selbst gehalten
    • Niemand wird dich wegen eines VPN- und OA-Kontos kontaktieren
    • Dynamische SMS-Verifizierung SMS wird nicht an andere weitergeleitet

  • Vorkehrungen für den Kundendienst

    • Mit wem unterhalte ich mich?
    • Prozess zur Bestimmung der Identität der anderen Partei
    • Welche Informationen kann ich zur Verfügung stellen?
    • Vertrauliche Operation Callback Verifikation Personalidentität

  • Physische Sicherheit

    • Zugangskontrolle
    • Shredder (um das Auslaufen kommerzieller Informationen zu verhindern)
    • Videoüberwachung
    • Verhindern Sie, dass Personal in den Arbeitsbereich gelangt

Sicherheitsoperation

  • Serversicherheit

    • MFA-Anmeldung
    • Einloggen mit Schlüssel
    • Deaktivieren Sie die Passwort-Anmeldung
    • Deaktivieren Sie nutzlose Konten
    • Kontosperrungsstrategie (um rohe Gewalt zu verhindern, ssh: fail2ban)
    • Sonderkonto prüfen (leeres Passwort)
    • Deaktivieren Sie die Remote-Remote-Anmeldung
    • Deaktivieren Sie nicht essentielle Serverdienste
    • Installieren Sie das IDS / IPS-System
    • Log-Sammlung

  • Aktualisierung

    • Die Systemversion sollte so nah wie möglich an der offiziellen Version sein.
    • Softwareversion (zum Beispiel: php) sollte rechtzeitig aktualisiert werden

  • log

    • Sammeln Sie umfassend
    • Echtzeitübertragung
    • Focus-Log wie ssh-Kernel-Log-Business-Protokoll
    • Business-Protokoll zum Desensibilisieren

  • Softwareinstallation

    • Nur vertrauenswürdige Softwarequellen sind erlaubt
    • Brauchen Sie eine neue Software für die Sicherheitsbewertung
    • Business Code sollte versuchen, nicht mit Root-Rechten zu laufen

  • Netzwerksicherheit

    • ACL beschränkt Geschäftskommunikation, die keinen gegenseitigen Zugriff erfordert
    • Sicherheitsgruppe sollte nicht angehängt werden
    • snmp ändert den Community-Namen

  • Datensicherheit

    • Eliminiere schwache Passwörter
    • Verbieten Sie anonyme Benutzer
    • Datenbankverschlüsselung (nach dem Prinzip der Minimierung von Berechtigungen)
    • Redis muss verschlüsselt sein
    • Big Data Services müssen eine Authentifizierung haben
    • MongoDB muss ein Passwort hinzufügen

  • Prozesssicherheitskontrolle

    • Zugriff auf Betrieb und Wartung
    • Öffnen des Domain-Namens
    • VPN-Anwendungsprozess
    • Springboard Bewerbungsprozess
  • Physische Sicherheit
    • Der Computerraumserver erlaubt keine USB-Schnittstelle
    • Das VGA-Display sollte sich im Ausgangszustand befinden

Produktmanager Sicherheitstraining

  • Offenlegung vertraulicher Informationen
  • Debug Schalter
  • Anti-schwere Unterschrift
  • http-DNS
  • Packet Hijacking
  • Sicherheit des Login-Mechanismus
  • Anti-Brute-Force-Rissbildung
  • Zeitfenster Rationalität
  • Sei kein SMS-Bomber
  • Jedes Benutzerpasswort zurückgesetzt
